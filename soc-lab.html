SOC Home Lab

A hands-on SOC environment I built to practice attack simulation (Kali/Hydra), detection and monitoring (Splunk), vulnerability management (Nessus), and digital forensics (Autopsy). This page documents the full journey: architecture, commands, troubleshooting, skills, and outcomes.

Overview

To strengthen my hands-on cybersecurity skills, I designed and implemented a Security Operations Center (SOC) home lab. The lab simulates a real-world enterprise environment, allowing me to practice attack simulation, detection, monitoring, vulnerability management, and forensics. It demonstrates the ability to build, troubleshoot, and operate a SOC environment from scratch using industry-standard tools.

Lab Architecture

  • pfSense — Firewall/gateway; controlled inbound/outbound traffic and resolved Kali connectivity.
  • Windows 10 — End-user workstation simulation.
  • Linux Server (Ubuntu) — Target with OpenSSH enabled for brute-force testing.
  • Windows Server (Domain Controller) — Active Directory.
  • Kali Linux — Attacker machine (Hydra).
  • Splunk Enterprise — SIEM for log collection/monitoring.
  • Nessus Essentials — Vulnerability scanning/reporting.
  • Autopsy — Forensic analysis of .E01 images.

Network & Troubleshooting

  • Configured pfSense firewall and NAT rules for controlled egress/ingress.

Attack Simulation (Red Team)

From Kali Linux, I simulated brute force and reconnaissance.

# Create test user on the Linux target
sudo adduser jeeva
sudo passwd jeeva  # set: 123456

# Hydra brute force (small list)
hydra -l jeeva -P /root/top10.txt ssh://10.0.10.22

# Hydra brute force (large list + verbose)
sudo gzip -d /usr/share/wordlists/rockyou.txt.gz   # if still gzipped
hydra -l jeeva -P /usr/share/wordlists/rockyou.txt ssh://10.0.10.22 -t 4 -V

# Port/service discovery pre-attack
nmap -sV -p- 10.0.10.22

Failed logins recorded in /var/log/auth.log on the target.

Log Monitoring (Blue Team)

Configured Splunk to ingest Linux authentication and system logs.

# Add inputs (do not publish real credentials in public repos)
sudo /opt/splunk/bin/splunk add monitor /var/log/auth.log \
  -sourcetype linux_secure -index main -auth 'admin:<redacted>'

sudo /opt/splunk/bin/splunk add monitor /var/log/syslog \
  -sourcetype syslog -index main -auth 'admin:<redacted>'

sudo /opt/splunk/bin/splunk restart

# SPL: view failed/successful SSH attempts
index=main sourcetype=linux_secure "Failed password"
index=main sourcetype=linux_secure "Accepted password"
index=main sourcetype=linux_secure jeeva

Challenges & Solutions

  • Kali had no internet → fixed via pfSense NAT/firewall rules.
  • Hydra wordlist missing → created/used top10.txt and rockyou.txt.
  • Splunk not showing logs → added auth.log/syslog inputs and restarted Splunk.
  • Autopsy empty evidence → reindexed the .E01 image and corrected settings.
  • OBS mic not recording → selected the external USB microphone in Windows & OBS.
