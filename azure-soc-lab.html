Azure Cloud SOC Lab

Microsoft Sentinel | Microsoft Defender XDR | Active Directory Attack Simulation

Overview

This project demonstrates the design and deployment of a cloud based Security Operations Center (SOC) in Microsoft Azure. The lab simulates a real enterprise security monitoring environment using Microsoft Sentinel and Defender XDR to detect and investigate security events generated from Active Directory and Windows endpoints.

The environment includes a domain controller, domain joined client machines, log ingestion through Azure Monitor Agent, security analytics using KQL, and automated incident response through Sentinel playbooks.

Lab Architecture

Components

  • Azure Virtual Network
  • Active Directory Domain Controller (DC01)
  • Windows 10 Client (WIN10)
  • Microsoft Sentinel (SIEM)
  • Microsoft Defender for Endpoint
  • Azure Monitor Agent
  • Log Analytics Workspace
  • Sentinel Analytics Rules
  • Logic App Playbooks

Network

  • Virtual Network
  • Subnet: 172.16.0.0/24
  • DC01 – Domain Controller
  • WIN10 – Domain Joined Endpoint

Lab 1 – Active Directory Deployment

Objective

Build a working Active Directory environment in Azure.

Implementation

  • Created Azure resource group: SOC-LAB-RG
  • Deployed Windows Server 2022 VM (DC01)
  • Installed Active Directory Domain Services
  • Configured DNS server
  • Domain created: soclab.local
  • Deployed Windows 10 VM and joined it to the domain

Validation

  • WIN10 → DC01 ping successful
  • DC01 → WIN10 ping successful
  • Client successfully joined SOCLAB.local domain

Lab 2 – Domain Authentication & Security Auditing

Objective

Enable Windows audit logging for security monitoring.

Configuration

  • Configured Advanced Audit Policies through Group Policy
  • Logon events monitoring
  • Account management tracking
  • Process creation logging
  • Kerberos authentication auditing

Important Event IDs

  • 4624 – Successful logon
  • 4625 – Failed logon
  • 4634 – Logoff
  • 4688 – Process creation
  • 4720–4738 – Account management

Lab 3 – Log Collection with Microsoft Sentinel

Objective

Ingest Windows security logs into Microsoft Sentinel.

Setup

  • Created Log Analytics Workspace: soc-lab-workspace
  • Connected Microsoft Sentinel to workspace
  • Installed Azure Monitor Agent on DC01 and WIN10
  • Configured Data Collection Rule for Windows Security Events

KQL Validation Queries

Heartbeat | sort by TimeGenerated desc
SecurityEvent | sort by TimeGenerated desc

Lab 4 – Simulated Attack (Brute Force)

Objective

Generate failed authentication attempts.

Attack Simulation

Multiple invalid RDP login attempts were performed which generated Windows Security Event ID 4625 (Failed logon).

Detection Query

SecurityEvent
| where EventID == 4625
| sort by TimeGenerated desc

Lab 5 – Sentinel Detection Rule

Objective

Detect brute force attacks.

SecurityEvent
| where EventID == 4625
| summarize FailedAttempts = count() by TargetUserName, IpAddress, Computer, bin(TimeGenerated, 5m)
| where FailedAttempts >= 5

Configured as a Scheduled Analytics Rule with a frequency of 5 minutes and lookback period of 30 minutes.

MITRE ATT&CK Tactic: Credential Access – Brute Force

Lab 6 – Incident Investigation

Objective

Investigate security incidents across Sentinel and Defender.

  • Opened Sentinel incident
  • Analyzed entities including user, host, and IP address
  • Pivoted to Microsoft Defender portal for device timeline investigation

Lab 7 – Automated Response (Playbooks)

Objective

Automate incident notifications.

  • Created Sentinel playbook using Logic Apps
  • Trigger: Sentinel alert generated
  • Action: Send automated email notification
  • Email includes alert name, severity, incident link, and entities

Lab 8 – Threat Hunting with KQL

Objective

Perform manual threat hunting using KQL queries.

Failed Logons

SecurityEvent
| where EventID == 4625
| summarize Attempts = count() by TargetUserName, IpAddress, bin(TimeGenerated, 1h)

Successful Logons

SecurityEvent
| where EventID == 4624
| summarize Logons = count() by TargetUserName

Privileged Logons

SecurityEvent
| where EventID in (4672,4648)
